# First Backend

Express.js + TypeScript backend application with MongoDB.

## Quick Start

```bash
# Install dependencies
pnpm install

# Development
pnpm dev

# Build
pnpm run build

# Test production build locally
pnpm run build && node --env-file=.env dist/index.js
```

## Environment Variables

Create a `.env` file with:

```env
FIRST_PORT=3000
FIRST_MONGODB_URI=mongodb+srv://...
FIRST_SECRET_KEY=your-secret-key
FIRST_URL_ORIGINS=http://localhost:3000,http://localhost:5173
FIRST_JWT_EXPIRES_IN=7d
FIRST_ENVIRONMENT=development
```

Or use unprefixed variables: `PORT`, `MONGO_URI`, `NODE_ENV`, etc.

## Tech Stack

- **Runtime:** Node.js v25.6+ (native `.env` support)
- **Framework:** Express.js
- **Language:** TypeScript with path aliases (`@Config/*`, `@Modules/*`, etc.)
- **Database:** MongoDB (Mongoose)
- **Deployment:** Vercel (serverless functions)

## Project Structure

```text
src/
├── index.ts               # Entry point - Simple Express app
└── constants/
    └── environments.ts    # Environment variables

dist/                      # Compiled output (generated by build)
api/index.js              # Vercel serverless wrapper
```

**Note:** Dependencies for MongoDB, Zod validation, middleware, etc. are
installed but not yet implemented.

## Development Notes

- Deployment uses compiled code from `dist/` directory
